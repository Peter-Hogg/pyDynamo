FiloType¶

class model.filoType.FiloType

Categorization for a point/branch based of Filo present. Note that order matters, 1->4 are counted as Filo.

ABSENT = <FiloType.ABSENT: 0>

Filo is not there.

BRANCH_ONLY = <FiloType.BRANCH_ONLY: 5>

Branch with no Filo.

BRANCH_WITH_INTERSTITIAL = <FiloType.BRANCH_WITH_INTERSTITIAL: 3>

Branch with downstream interstitial Filo.

BRANCH_WITH_TERMINAL = <FiloType.BRANCH_WITH_TERMINAL: 4>

Branch with downstream terminal Filo.

INTERSTITIAL = <FiloType.INTERSTITIAL: 1>

Filo in the middle of a branch.

TERMINAL = <FiloType.TERMINAL: 2>

Filo at the end of a branch

History¶

class model.history.History(liveState)

Respresents the current live active state of the app, as well as a history of changes. After snapshotting with pushState(), the app can walk through history with undo/redo.

liveState = None

State singleton that is being modified by the app.

pushState()

Remember the current live state so we can go back to it later if needed.

redo()

Undo an undo by reverting the live stack to a recently undone state.

redoStack = []

After undo, keep previous state in redo stack so undo can be redo’d.

undo()

Revert live state back to the most recent pushed state.

undoStack = []

List of previous state snapshots that can be undo’d to.
